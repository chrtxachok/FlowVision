# 📋 Отчет о проекте FlowLogix OCR Service

## 1. Обзор проекта

**FlowLogix OCR Service** — это микросервис для распознавания и извлечения структурированных данных из документов логистики (накладные, счета-фактуры, акты).

### Ключевые возможности:
- ✅ Распознавание русского языка и английского
- ✅ Извлечение структурированных данных из документов
- ✅ Поддержка форматов: **JPEG, PNG, PDF**
- ✅ Точность >85% для стандартных накладных
- ✅ **Два движка OCR на выбор**:
  - **EasyOCR** (основной, рекомендуется)
  - **PaddleOCR** (legacy, автоматический fallback)

### Требования:
- Python 3.11 - 3.12 (критично!)
- Минимум 2 ГБ оперативной памяти
- Docker (опционально)

## 2. Архитектура приложения

### 2.1 Общий поток данных (Pipeline)

```
Фронтенд (браузер)
    ↓
    ├─→ Загрузка файла (drag-and-drop или выбор)
    ├─→ Выбор типа документа (waybill/invoice/act)
    ├─→ Отправка на API
    │
Бэкенд (FastAPI)
    ↓
    ├─→ Валидация (размер, формат, API ключ)
    ├─→ Преобразование файла в изображение
    ├─→ Улучшение качества изображения
    │
OCR Pipeline
    ↓
    ├─→ **Выбор backend'а**:
    │   ├─→ EasyOCR (Reader.readtext) ← обычно используется
    │   └─→ PaddleOCR v3 (OCR.detect + OCR.recognize)
    ├─→ Распознавание текста с confidence
    ├─→ Нормализация результатов
    │
Постобработка (Postprocess)
    ↓
    ├─→ Преобразование raw-результата в стандартный формат
    ├─→ Выделение отдельных строк с координатами
    │
Процессоры документов (Document Processors)
    ↓
    ├─→ Применение regex-паттернов для извлечения полей
    ├─→ Вычисление confidence для каждого поля
    ├─→ Структурирование данных (WaybillData, InvoiceData)
    │
Фронтенд (браузер)
    ↓
    └─→ Отображение результатов с extracted_data, raw_text и JSON
```

## 4. OCR Pipeline (движок обработки)

### 4.1 Выбор OCR backend'а

**Конфигурация (app/config.py):**

### 4.2 Этап 1: Преобработка (Preprocess)

**Файл:** `app/ocr/preprocess.py`

#### Обработка разных форматов:
```
JPG/PNG → PIL.open() → cv2.cvtColor(RGB→BGR) → np.ndarray
PDF → PyMuPDF.render() → первая страница → BGR
```

#### Улучшение качества:
```python
def enhance_image(img):
    # 1. Контрастирование (CLAHE)
    # 2. Бинаризация (adaptive thresholding)
    # 3. Шумоподавление (morphological operations)
    # 4. Масштабирование (если изображение слишком маленькое)
    return enhanced_img
```

### 4.3 Этап 2: Распознавание текста (OCR)

#### EasyOCR (рекомендуется):
```python
from easyocr import Reader

reader = Reader(['ru', 'en'], gpu=False)
results = reader.readtext(image_rgb)  # [(bbox, text, confidence), ...]
```

**Преимущества EasyOCR:**
- ✅ Легкая установка (pip install easyocr)
- ✅ Лучше с русским языком
- ✅ Автоматическая загрузка моделей
- ✅ Работает на CPU
- ✅ Быстрее при первом запуске

#### PaddleOCR v3 (fallback):
```python
from paddleocr import PaddleOCR

ocr = PaddleOCR(
    use_doc_orientation_classify=False,
    use_doc_unwarping=False
)
results = ocr.ocr(image_bgr)
```

**Когда используется:**
- Если EasyOCR не установлен
- Для документов со сложной структурой
- Если нужна высокая точность ценой производительности

### 5.2 WaybillProcessor (Обработчик накладных)

**Файл:** `app/processors/waybill.py`

#### Извлекаемые поля:

| Поле | Regex-паттерны | Confidence | Пример |
|------|---|---|---|
| **waybill_number** | `накладная\s*№?\s*([а-я0-9/-]+)` | 0.90 | ТТН-2024-001 |
| **date** | `дата[:\s]+(\d{1,2}[./-]\d{1,2}[./-]\d{2,4})` | 0.85 | 01.06.2024 |
| **sender** | `отправитель[:\s]+([а-яё0-9\s,.-]+?)` | 0.80 | ООО Логистика |
| **recipient** | `получатель[:\s]+([а-яё0-9\s,.-]+?)` | 0.80 | ИП Иванов |
| **cargo_description** | `груз[:\s]+([а-яё0-9\s,.-]+?)` | 0.75 | Электроника |
| **cargo_mass_kg** | `масса[:\s]+(\d+(?:[.,]\d+)?)\s*кг` | 0.90 | 45.5 |
| **total_amount** | `итого[:\s]+(\d+(?:\s\d{3})*(?:[.,]\d{2})?)` | 0.90 | 15000.00 |
| **driver_name** | `водитель[:\s]+([а-яё\s]+?)` | 0.70 | Сидоров Петр |
| **vehicle_number** | `[а-я]{1}\d{3}[а-я]{2}\d{2,3}` | 0.85 | А123БВ77 |

#### Логика обработки:

```python
class WaybillProcessor(BaseProcessor):
    def process(self, ocr_result, image_type):
        full_text = ocr_result['full_text'].lower()  # ← Ключевой момент!
        lines = ocr_result['lines']
        
        # Применяем regex-паттерны для каждого поля
        extracted_data = {
            "waybill_number": self._extract_waybill_number(full_text, lines),
            "date": self._extract_date(full_text, lines),
            ...
        }
        
        # Вычисляем среднее confidence
        confidence = mean([v['confidence'] for v in extracted_data.values()])
        
        return {
            "document_type": "waybill",
            "confidence": confidence,
            "extracted_data": extracted_data,
            "raw_text": ocr_result['full_text'],
            "metadata": {
                "fields_found": count(найденных полей),
                "processing_method": "EasyOCR" или "PaddleOCR"
            }
        }
```

    // Проверка типа
    if (!['image/jpeg', 'image/png', 'application/pdf'].includes(file.type)) {
        showError('Неподдерживаемый формат');
        return;
    }
    
    // Проверка размера (10 МБ)
    if (file.size > 10 * 1024 * 1024) {
        showError('Файл слишком большой');
        return;
    }
    
    selectedFile = file;
    displayFileInfo(file);
}

#### 3️⃣ **Отправка на сервер**
```javascript
async function processDocument() {
    const formData = new FormData();
    formData.append('file', selectedFile);
    formData.append('document_type', document.getElementById('documentType').value);
    formData.append('api_key', document.getElementById('apiKey').value);
    
    showLoading();  // Показываем спиннер
    const startTime = Date.now();
    
    try {
        const response = await fetch('/api/v1/ocr/process', {
            method: 'POST',
            body: formData,
            timeout: 120000  // 2 минуты
        });
        
        const data = await response.json();
        const processingTime = Date.now() - startTime;
        
        displayResults(data, processingTime);  // Показываем результаты
        
    } catch (error) {
        showError(error.message);
    }
}
```

## 11. Производительность и оптимизация

### Время обработки (típичное):
- **Преобработка:** 50-100 мс
- **Распознавание текста (EasyOCR):** 800-2000 мс
- **Постобработка:** 50-100 мс
- **Применение процессора:** 50-200 мс
- **Всего:** 950-2400 мс (0.95-2.4 секунды)

### Оптимизация:
1. **Кеширование моделей:** Модели загружаются один раз в памяти
2. **GPU ускорение:** Установите `OCR_DEVICE="cuda"` для NVIDIA GPU
3. **Батчевая обработка:** Возможна обработка нескольких изображений за раз
4. **Асинхронность:** Фронтенд показывает прогресс во время загрузки

## 12. Известные ограничения и улучшения

### Текущие ограничения:
1. ❌ **Regex-based parsing:** Использует жесткие regex-паттерны (не ML-based)
2. ❌ **Работает только с русским:** Проблемы с документами на других языках
3. ❌ **Накладные структура:** Оптимизирован для стандартных ТТН
4. ❌ **Таблицы не выделяются:** Не разделяет табличные данные
5. ❌ **Изображения с углом:** Не исправляет повернутые документы

### Планируемые улучшения:
- ✨ Интеграция **Donut** (End-to-End Document Understanding) для лучшего понимания структуры
- ✨ **Нейросетевые экстракторы** вместо regex
- ✨ **Табличное разделение** (Detection of table structure)
- ✨ **Поддержка других языков** (EN, DE, FR)
- ✨ **Исправление поворотов** (Document unwarping)
- ✨ **Кеширование результатов** (Redis)
- ✨ **Мониторинг и метрики** (Prometheus, Grafana)

## 15. Часто задаваемые вопросы

### Q: Почему не устанавливается EasyOCR?
**A:** Убедитесь, что Python 3.11-3.12. На более новых версиях могут быть проблемы совместимости.

### Q: Как улучшить качество распознавания?
**A:** 
1. Загружайте изображения высокого качества (200 DPI)
2. Избегайте сильно повернутых документов
3. Убедитесь, что документ полностью на изображении
4. Используйте контрастные изображения

### Q: Почему confidence низкий?
**A:** 
1. Изображение плохого качества
2. Текст нечеткий или повернут
3. Документ содержит рукописный текст
4. Используется неправильный OCR backend

### Q: Можно ли обрабатывать несколько страниц PDF?
**A:** Текущая версия обрабатывает только первую страницу. Для всех страниц нужно модифицировать `preprocess.py`.

### Q: Как изменить типы поддерживаемых документов?
**A:** Создайте новый процессор в `app/processors/your_document.py` и зарегистрируйте в `OCRService._load_processors()`.
